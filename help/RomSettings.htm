Rom Settings

Overview

In the Rom Settings, you configure all the things, that the Rom Fumbler needs to know about your Rom.
All this is needed to create icons in the correct size and finally build a flashable-zips that fits your Rom!

Learn how to find the right Rom Settings

Rom Settings can be saved and loaded from config-files.

And new since Version 25.0 Rom Settings can be exported into a Rom Preset (for details see next paragraph)

Rom Presets

Each Rom has its specialties. For example: Roms on Galaxy S2 are normally hdpi (exception Ressurrection Remix --> xhdpi).
Roms on Galaxy S3 are xhdpi and on Galaxy S4 xxhdpi! Filename-patterns are slightly different and so on.

So, Rom Presets bundle all these settings that need to be done for a special Rom. And you can apply this bunch of settings just by choosing the right Rom Presets for your Rom!

If your Rom is not in the Dropdownbox...you have two options...

1. Set up everything manually (Learn how to find the right Rom Settings)
2. add your own Rom Presets

Howto add your own RomPresets:

1. the smart way..

Set up the Rom Settings you need and export them to a new Rom Preset! (Use the "export save-button"...the one with the red arrow)
Restart the Romfumbler after that to have the new Rom Preset available in the dropdownbox!

2. the manual way...

Rom Presets are stored in the subfolder ./rompresets/<presetname>.
These files are plain textfiles!

  • Open an existing file with a simple texteditor,
  • edit all entries
  • and save it into a new file!
  • Make sure you change the field "RomName" to a unique name!
  • Restart the Rom Fumbler and you will find your new Rom Preset in the dropdownbox!

Hint:
If you want the Rom Fumbler to start with a special Rom Preset every time, then go to "Global Settings" and choose your Rom Preset there.
With the next start of the Rom Fumbler, everything will be configured for your Rom.

Request:
If you have a working Rom Preset for a Rom not yet supported...please upload the textfile to the Rom Fumblers XDA Thread!
I will then make it available for everybody by adding it to the next Release!

Main Rom Settings

Here you can choose different templates that are used to create flashable-zips. There are two different kind of templates.

  1. UniversalFlasherTool (UTF)
    + The UTF Templates create a flashable-zip as backup. In case you dont like the new icons or something went wrong, you can simply flash the backup.zip!
    + You find theses backup.zips on your internal or external sdcard in the subfolder /UniversalFlasher/...
    - the templates are big (around 2MB)
    - they might have problems flashing Samsung Stock Roms with /preload partition!
  2. Villian Theme System (vtheme) (template-names must contain "vrtheme")
    + small template file (0.6MB)
    - no flashabel Backup...it just backups the apks into a subfolder of your sdcard ./vrtheme-backup/...
I recommend using UTF Templates!

Howto add your own templates:

  • goto the subfolder ./template of the Romfumbler
  • and put your zip in that folder too
  • the  name should contain "vrtheme" if it is a vrtheme-template
  • and the name should contain "preload" if it is supposed to flash to Samsung stock roms with preload partition.
  • if you follow those naming conventions, the checkboxes for vrtheme and preload will be set accordingly and the Morphpath will be adjusted automaticly!

 What is a MorphPath ?

The Morphpath is the path within the flashable zip, where your icons are stored!
Depending on the template-type the path is a little bit different.
To generate the Morphpath fitting the choosen template-type...there is the checkbox "is VRTheme Template"
...click it and you will see how the Morphpath changes!

The Checkbax "is VRTheme Template" is set automatically, if the template-name contains the substring "vrtheme" (see Howto add your own templates)

What else submits to the Morphpath ?

On Galaxy S2 Samsung Stock Roms (and custom Roms based on Samsung Stock Rom) there is a specialty!
Samsung invented a new partition /preload ...they should have named ist /bloatload...because the /system partition has grown so much, that they needed more space! So they "invented" the /preload partition and changed the path to the system/apps :-(

So if you click the "Rom uses Preload Partition" checkbox...the Morphpath is set accordingly.
The Checkbax "Rom uses Preload Partition" is set automatically, if the template-name contains the substring "preload" (see Howto add your own templates)

Attention: So far only vrtheme templates can flash Preload-Roms!

Is that all for Morphpath ?

Almost ! ... there are the "drawable" comboboxes...they define the folder under the Morphpath where the icons are actually stored within the zip and later within the apk.

So the Morphpath + drawable are the complete path to the icons within the zip!

Why "Morph" ?

Morph means that the icons are morphed into the target apk's on your Rom. Apk's are like zips (check it out and open an apk with Winzip or 7zip!)
During flashing the icons within the target apk are replaced by those stored in the Morphpath. The rest of the apk is untouched!

This has a big advantage! The flashable zips created with the Rom Fumbler are compatible to your Rom, even if you update your Rom with a new version. Just flash the Zip again!
As long the folder- and filenaming-structure within the target apk is not changed the flashable zips are always compatible and can be flashed to the new version of your Rom as well!

If you look around at XDA you find a lot of people sharing flashable-zips as "Themes" that contain a complete SystemUI.apk...
You can imagine, that you need to have the exact Rom-version from where this SystemUI has been taken and modified.
If not, you are doomed and will probably end up with a bootlooping device!

This can not happen with Zips created by the Rom Fumbler!!!!

What can go wrong with the Morphpath ?

Well the worst that can happen, is that you dont see your icons after flashing!
This has probably the following reasons:

  • you chose the wrong drawable folder
  • you are on a "Preload Rom" (so far only known on Galaxy S2 Stock) and  used a non preload template!
  • you are on a "Preload Rom" and used an UTF template (preload or not...I still need to figure out how to use UTF templates on "Preload Roms")
